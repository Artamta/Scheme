# CTfS Assignment II - Short Report

## 1. What is implemented

I implemented a simple molecular dynamics simulator in Scheme using the Lennard-Jones 6-12 pair potential:

$$
V(r)=4\epsilon\left[\left(\frac{\sigma}{r}\right)^{12}-\left(\frac{\sigma}{r}\right)^6\right]
$$

The force is computed as:

$$
F(r)=-\frac{dV}{dr}
$$

I reused Assignment 1 symbolic differentiation (`deriv` and `simplify`) to build the force expression once, and then evaluate it numerically during simulation.

For time integration, I used Verlet position update:

$$
r(t+\Delta t)=2r(t)-r(t-\Delta t)+a(t)\Delta t^2
$$

and velocity estimate:

$$
v(t)=\frac{r(t+\Delta t)-r(t-\Delta t)}{2\Delta t}
$$

## 2. Abstraction layers

1. **Model definition layer** (`src/model_defination.scm`)
   - Stores species masses and LJ pair parameters.

2. **Physics layer** (`src/model_physis.scm`)
   - Contains symbolic LJ expression, force calculation, vector operations, and force accumulation.

3. **Integrator + driver layer** (`src/main.scm`)
   - Creates particle records, performs step-by-step simulation, and writes CSV output.

This separation made the code easy to debug and extend.

## 3. Required conceptual answers

### Q1) What would you change for multi-species systems?

Only the parameter tables need extension.
- Add more entries to mass table.
- Add more pair entries to LJ parameter table.

The integration loop does not need to change.

### Q2) What would you change for different potentials?

Change potential expression in the physics layer and regenerate force with symbolic differentiation.
The rest of the simulation process (force sum, acceleration, Verlet update) can remain the same.

### Q3) How to incorporate Assignment 1 symbolic differentiation?

Already done in this submission.
- `asg1/simplify.scm` and `asg1/deriv.scm` are loaded.
- LJ potential is represented symbolically.
- Force expression is generated by `-dV/dr` symbolically.
- Force is evaluated numerically at runtime for each pair distance.

### Q4) Comment on analytical vs computational process view

Analytical methods aim for closed-form expressions that directly describe behavior.
Computational methods model behavior as a process over time. The state is updated repeatedly from local rules.

In this assignment, Verlet integration clearly shows process view: each new state depends on previous states and current acceleration. This makes simulation practical even when closed-form solutions are hard or unavailable.
